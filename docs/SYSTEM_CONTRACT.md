# Hirobius Design System — System Contract

> This document defines scope, stability, ownership, and consumption rules for
> the Hirobius Design System (HDS). It is the authoritative reference for what
> is committed, what is experimental, and who can make breaking changes.

Last updated: 2026-05-02. Unit: 10n-8-system-contract-doc.

---

## Stable Surface

The following artifacts are **stable** and subject to semver discipline.
Consumers (Figma plugin, portfolio site, future client deployments) may rely on
these without expecting silent breakage.

| Artifact | Path | Stability |
|----------|------|-----------|
| Design tokens | `hirobius.tokens.json` | Stable |
| Generated CSS vars | `src/styles/tokens.css` | Stable |
| Generated TS constants | `src/app/design-system/generated-tokens.ts` | Stable |
| Manifest schema | `manifest/schema.json` | Stable (semver-guarded) |
| HDS Manifest | `public/hds-manifest.json` | Stable (generated, schema-locked) |
| Primitive components | `HdsButton`, `Input`, `Card`, `Dialog`, … | Stable |
| LLM index | `public/llms.txt` | Stable (generated) |

**Stable means:** removing a field, renaming a token path, or changing a required
prop is a **breaking change** that requires an explicit migration step and must
be announced in the commit message and ledger.

## Breaking Change Policy

A change is **breaking** if it:

1. Removes or renames a token path that has a CSS var (`--hds-*`) counterpart
2. Removes a required field from `manifest/schema.json#required`
3. Removes a property definition from `manifest/schema.json#properties`
4. Removes a public component export from `src/index.ts`
5. Changes a primitive component's required prop name or type
6. Removes an enum value from a stable prop (e.g. `tier`, `stability`)

**To ship a breaking change:**

1. Add a T3 ledger entry in `docs/logs/AI_DECISION_LEDGER.md`
2. Bump the semantic version in the affected artifact (schema.json, package.json)
3. Update `manifest/schema.lock.json` (`node scripts/check-manifest-schema-semver.mjs --update-lock`)
4. Update any callers in the same commit
5. Note the migration path in the commit message

## Who Can Merge

- **Autonomous agents** may merge: T2 build units (additive, non-breaking), doc
  updates, test additions, validator additions in warn-only mode.
- **Requires Adrian review**: T3 units (architectural decisions), any breaking
  change, schema version bumps, dependency additions.
- **Hard-blocked**: pushing to `main`/`master` directly, running `check:release`,
  deploying to production — these require explicit Adrian instruction.

## Auto-Generated Files — Do Not Edit Directly

The following files are regenerated by build scripts. Direct edits are
overwritten on the next `pnpm tokens` or `pnpm manifest:generate` run:

| File | Generator |
|------|-----------|
| `public/llms.txt` | `scripts/generate-llms-txt.mjs` |
| `DESIGN.md` | `scripts/build-design-md.mjs` |
| `DESIGN-HANDOFF.md` | `scripts/build-handoff.mjs` |
| `public/hds-manifest.json` | `scripts/generate-manifest.mjs` |
| `src/styles/tokens.css` | `scripts/build-tokens.mjs` |
| `src/styles/tokens.generated.css` | `scripts/build-tokens.mjs` |
| `src/app/design-system/generated-*.ts` | `scripts/build-tokens.mjs` |
| `tailwind.config.tokens.cjs` | `scripts/build-tokens.mjs` |

To update these, edit their template/source and re-run the generator.

## v1 Gate Checklist

The system is considered v1 when:

- [ ] All stable surface entries pass schema validation in CI
- [ ] `manifest/schema.lock.json` is committed and guarded (`12n-api-manifest-schema-semver`)
- [ ] At least one external consumer (Figma plugin) reads from stable endpoints
- [ ] This document is linked from `public/llms.txt` under `agent-context`
- [ ] Breaking change policy has been exercised at least once (documented in ledger)
