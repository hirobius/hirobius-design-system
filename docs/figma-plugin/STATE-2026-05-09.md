## 3. Verified Figma-vs-tokens drift (from `pnpm figma:audit --strict`)

The script correctly reports the same drift tracked in
`12q-figma-system-drift`. **None of these are script bugs — they are real
data drift between `hirobius.tokens.json` and `hirobius.figma-variables.json`
/ `hirobius.figma-variables-api.json`:**

- **Hirobius/Primitives**: 8 missing — `radius/12`, `shadow/color`,
  `typography/family/display`, `typography/letterSpacing/caps`,
  `typography/lineHeight/{relaxed,snug}`, `typography/size/7xl`,
  `typography/weight/light`.
- **Hirobius/Semantic**: 5 missing — `color/border/subtle`,
  `docs/space/{inline,paragraph,section}`, `typography/lineHeight/none`.
- **Hirobius/Component**: 18 missing button-size vars
  (lg/md/sm × 6 each: fontSize, height, iconSize, itemSpacing, paddingX,
  paddingY); 3 extras: `button/{fontSize,fontWeight,minWidth}`.
- **REST payload**: 257 actual vs 305 expected variables; 3 collections
  vs 3 expected (count of collections OK).

## 4. HITL decisions Adrian must make

### HITL #1 — Figma vs tokens canonicity (BLOCKS DoD #1, #4, #7)

Pick the canonical source for the design-system variables. Both options
fully resolve `12q-figma-system-drift` and unblock the umbrella task.

**Option A — Tokens are canonical, regenerate Figma master.**
- Run `pnpm figma:library:generate` (or `pnpm ui:masters` + `pnpm sync`)
  to push the missing 8 / 5 / 18 vars into Figma and remove the 3 extras.
- This is the *expected* direction per current docs (token source is
  ground truth, Figma mirrors it). Lowest-risk.
- Effort: ~1 h once Adrian okays it (generation is automated; manual
  verification of the regenerated Figma file required).
- Side effect: any in-Figma edits to those 26 variables since they
  drifted will be overwritten. Likely zero loss because the drift is
  schema gaps, not value tweaks — but Adrian should eyeball the
  `extras` (button/{fontSize,fontWeight,minWidth}) before regen.

**Option B — Figma is canonical, adjust hirobius.tokens.json.**
- Add the 3 extras (`button/fontSize`, `button/fontWeight`,
  `button/minWidth`) to tokens; remove the 26 missing entries from
  tokens or accept they're "not in Figma yet."
- Higher-risk: tokens may be referenced by React components / the
  manifest / DESIGN.md. Removing live tokens cascades through
  `pnpm sync`, `verify-tokens.mjs`, `check-token-renames.mjs`.
- Effort: ~2-3 h plus regression sweep.
- Recommended only if there's a deliberate reason the Figma file diverged
  (e.g. design decisions Adrian made in Figma that haven't been
  back-propagated).

**Default recommendation:** Option A unless Adrian remembers
intentionally editing those button-size variables in Figma.

### HITL #2 — Code Connect mappings (does not block, but unlocks DoD #4 fully)

`figma-parity-check.mjs` reports all 56 `componentSpecs` entries have
`figmaLink: null` (or empty / TODO). Without real Figma node IDs the
generator falls back to placeholder frames per FIGMA_BRIDGE.md §5 — the
sample layout in DoD #4 will succeed, but it won't be using the
real published library, just structurally compatible scaffolds.

To wire Code Connect properly, Adrian needs to either:
- Author the mappings manually in `componentSpecs[].figmaLink`, or
- Use `mcp__claude_ai_Figma__add_code_connect_map` once the Figma file
  has stable node IDs (post-Option-A regen).

This is mechanical work post-HITL-#1; not a blocker.
