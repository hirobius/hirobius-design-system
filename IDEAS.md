# IDEAS

Exploratory ideas for the Hirobius portfolio and design-system product.

This file is intentionally separate from `TASKS.md` and `BACKLOG.md`.

- `TASKS.md` is active launch work.
- `BACKLOG.md` is committed roadmap work.
- `IDEAS.md` is for promising concepts that still need validation, clearer user value, or release timing.

Move an idea into `BACKLOG.md` only when it has:

- a clear audience or workflow
- a reason it belongs in the product story
- a validation path
- an owner or next concrete pass

Each idea below carries one source tag: `[Foundations]`, `[Components]`, or `[Architecture]`.

## AI Workflow Ideas

- AI changelog generator - generate structured changelogs from git history grouped by change type. [Architecture]
- Natural language token search - search for tokens with phrases like "muted dark background". [Foundations]
- Getting Started with AI prompts - prompt blocks for scaffolding components, changelogs, audits, and token suggestions. [Architecture]
- Code block component - reusable syntax-highlighted code block for guides and prompts. [Components]

## Interactive Showcase Ideas

- Live token inspector - click a rendered element and inspect resolved tokens in a popover. [Foundations]
- Token playground - tweak a token and watch the system update live. [Foundations]
- Multi-brand theming demo - swap one brand token object to produce a clearly different look. [Foundations]

## Adoption Ideas

- Public starter kit - package the smallest useful HDS setup for a separate demo app. [Architecture]
- Case-study evidence gallery - collect screenshots, diffs, audits, and design decisions into reusable proof blocks. [Architecture]
- Design-system scorecard - summarize docs coverage, token health, component readiness, and Figma parity in one executive view. [Architecture]
