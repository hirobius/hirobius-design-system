# UI/UX Pro Max — Design Intelligence (Helper Mode)

> **Project Context Override — READ FIRST**
>
> This skill is a **UX reference and checklist layer only** — not a design system generator. It contains no brand values. For all visual decisions on Hirobius / Adrian Milsap projects, the authoritative sources are:
>
> | Decision area | Read this |
> |---|---|
> | Color (all — primitives, semantic, component) | `DESIGN.md` → **Colors** section |
> | Typography (typeface, ramp, weights, line-height) | `DESIGN.md` → **Typography** section |
> | Spacing scale | `DESIGN.md` → **Spacing** section |
> | Border radius rules | `DESIGN.md` → **Corner-Radius Policy** section |
> | Motion / easing / duration | `DESIGN.md` → **Motion** section |
> | Component-level tokens | `DESIGN-HANDOFF.md` → **Component Tokens** section |
> | Hard constraints (button corners, accent color, font lock) | `DESIGN.md` → **Do's and Don'ts** section + global `CLAUDE.md` → **Design Agent Context** |
> | How to consume tokens in code | Global `CLAUDE.md` → **Design Token Pipeline → Consuming tokens in code** |
>
> **Never use** color palettes, font pairings, or spacing scales generated by this skill's search script. Use those outputs for **UX pattern and interaction guidance only** — then map any directional intent (e.g. "high-contrast, minimal") to the nearest token in DESIGN.md.
>
> This skill has **no raw values by design.** If it ever references a specific hex, px value, or typeface name, that is a bug — defer to DESIGN.md instead.

## When to Apply

**Must use** for:
- Designing new pages or screens (landing pages, dashboards, admin panels)
- Creating or refactoring UI components
- Reviewing UI code for UX, accessibility, or visual consistency
- Implementing navigation structures, animations, or responsive behavior
- Making product-level design decisions (information hierarchy, interaction model)

**Skip** for:
- Pure backend logic, API design, infrastructure, non-visual scripts
- Token pipeline work (use DESIGN.md + DESIGN-HANDOFF.md + hirobius.tokens.json instead)

## 10 Rule Categories (Priority Order)

| Priority | Category | Impact |
|---|---|---|
| 1 | Accessibility | CRITICAL — contrast 4.5:1, focus states, alt text, keyboard nav |
| 2 | Touch & Interaction | CRITICAL — 44×44pt targets, 8px+ spacing, loading feedback |
| 3 | Performance | HIGH — WebP/AVIF, lazy loading, CLS < 0.1 |
| 4 | Style Selection | HIGH — match style to product type, consistency, SVG icons only |
| 5 | Layout & Responsive | HIGH — mobile-first, viewport meta, no horizontal scroll |
| 6 | Typography & Color | MEDIUM — 16px base, 1.5 line-height, semantic tokens |
| 7 | Animation | MEDIUM — 150–300ms, transforms only, respect reduced-motion |
| 8 | Forms & Feedback | MEDIUM — visible labels, error placement, progressive disclosure |
| 9 | Navigation Patterns | HIGH — bottom nav ≤5, predictable back, deep linking |
| 10 | Charts & Data | LOW — chart type matching, accessible colors, tooltips |

## Pre-Delivery Checklist

**UX / Accessibility (this skill)**
- [ ] No emojis as structural icons — SVG only (see CLAUDE.md → Icons)
- [ ] All touch targets ≥44×44pt with visible pressed feedback
- [ ] Text contrast ≥4.5:1 in both light and dark modes
- [ ] Safe area compliance (notch, gesture bars, home indicator)
- [ ] Reduced-motion respected
- [ ] Forms: visible labels, inline error placement, submit feedback

**HDS token compliance (DESIGN.md is authoritative)**
- [ ] Spacing rhythm — see DESIGN.md → Spacing
- [ ] Border radius rules — see DESIGN.md → Corner-Radius Policy
- [ ] Semantic color tokens used — no hardcoded values
- [ ] Typography via correct method — see CLAUDE.md → Consuming tokens in code
- [ ] Font, accent color, button corners — see DESIGN.md → Do's and Don'ts
