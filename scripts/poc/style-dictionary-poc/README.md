# Style Dictionary POC — Hirobius Design System

## What this proves

Style Dictionary v5 can emit semantically-equivalent CSS custom properties for
`primitive.color`, `primitive.space`, and `primitive.radius` from `hirobius.tokens.json`
using 3 custom transforms and 1 custom format (~60 lines total).

Run the parity check:
```sh
bash scripts/poc/style-dictionary-poc/run.sh
# or:
pnpm audit:style-dictionary
```

Expected output: `RESULT: PASS` with zero diff lines.

## Files

| File | Purpose |
|---|---|
| `sd.config.mjs` | Style Dictionary config with 3 custom transforms + 1 format |
| `expected.css` | Snapshot extracted from `src/styles/tokens.css` for POC scope |
| `actual.css` | Generated by SD (gitignored from tracking but useful for local inspection) |
| `run.sh` | Runs SD, normalizes both files, diffs them. Exits 0 on parity. |

## What is proven (pass)

64 CSS custom properties covering:
- `primitive.color.*` — neutral ramp, blue ramp, red/green/amber feedback, projectBrand colors
  - Includes OKLCH color strings (SD passes them through as-is)
  - Includes camelCase path segment names (via `name/hds-preserve-camel` transform)
- `primitive.space.*` — 4px-grid spacing scale with DTCG `{value, unit}` dimension objects
- `primitive.radius.*` — corner radius scale

Custom transforms required and already implemented in `sd.config.mjs`:

1. **`name/hds-preserve-camel`** — SD's default `name/kebab` would convert
   `projectBrand.microsoftGameDev` → `project-brand-microsoft-game-dev`. build-tokens.mjs
   preserves original key casing (`projectBrand-microsoftGameDev`). This transform
   joins `token.path` with `-` without case conversion.

2. **`value/hds-dimension-px`** — DTCG uses `{value: N, unit: 'px'}` objects for dimensions.
   SD's built-in `size/px` only handles unitless numbers. This transform converts the object
   format to `Npx` strings, including zero-value `0px`.

3. **`hds/css-variables` format** — SD's built-in `css/variables` appends inline `/** comment */`
   per token from `$description` fields. build-tokens.mjs emits no inline comments. The custom
   format strips comments from output.

## What is NOT proven (out of scope for this POC)

The following require additional custom transforms/formats — detailed in the migration plan:

| Feature | Gap | Effort |
|---|---|---|
| `primitive.duration` | SD emits `[object Object]` for `{value,unit}` duration objects | Small: same fix as dimension |
| `primitive.easing.elastic` (spring type) | Custom `$type: spring` → SD emits `[object Object]` | Small: custom transform `spring(stiffness, damping, mass)` |
| `semantic.motion.*` | Composite expansion: one token → 2 CSS vars (`--hds-motion-X-duration/easing`) with renamed prefix | Medium: custom format block |
| `semantic.typography.*` | Composite expansion: one token → 5–7 CSS vars + optional `maxWidth`/`textTransform` | Medium: custom format block |
| `semantic.elevation.*` | Composite expansion: one token → 3 CSS vars (`-surface/-shadow/-border`) | Medium: custom format block |
| Dark mode block | `[data-theme="dark"]` with per-token Dark mode values from `$extensions.com.figma.variables.modes.Dark` | Medium: custom format + SD doesn't have a native "modes" concept |
| Tenant overlay | `[data-tenant="slug"]` blocks from `tenants/*/tokens.json` | Large: custom plugin or separate SD run per tenant |
| TypeScript files | `generated-tokens.ts`, `generated-token-refs.ts`, `generated-token-vars.d.ts`, etc. | Large: custom formats; not SD's core competency |
| `hds-manifest.json` | Deep component-spec metadata (not a token output) | Out-of-band: stays as separate script |
| `tailwind.config.tokens.cjs` | Role-to-Tailwind color/shadow/radius mapping | Medium: custom format or out-of-band script |

## Technical notes

- Style Dictionary v5 uses `usesDtcg: true` to parse `$value`/`$type` fields
- `outputReferences: true` preserves `var(--primitive-...)` chains for semantic tokens
  (without it, SD resolves aliases to raw values, breaking the dark-mode cascade)
- SD v5 format hooks must be plain async functions, not `{format: fn}` objects
- SD v5 transform hooks follow `{type, filter?, transform}` shape
- The `name/kebab` built-in is destructive for camelCase token keys; always override
  with `name/hds-preserve-camel` for this codebase
